# Wind金融终端自动化 - 性能优化指南

## 📊 性能瓶颈分析

### 为什么操作这么慢？

Midscene 的每次操作都需要以下步骤：

```
1. 截取屏幕 → 2-3秒
2. 发送到 AI 模型 → 1-2秒
3. AI 分析图像并定位元素 → 5-15秒（取决于模型性能）
4. 返回坐标并执行操作 → 1秒
5. 等待界面响应 → 1-3秒
```

**单次操作总耗时：10-25秒**

---

## 🚀 性能优化方案

### 方案 1：启用缓存（推荐）⭐⭐⭐⭐⭐

**原理**：首次运行时记录 AI 决策，后续运行直接复用，无需 AI 分析。

**配置**：
```env
# .env 文件
MIDSCENE_CACHE=true
MIDSCENE_CACHE_MODE=read-write
```

**效果**：
- 首次运行：正常速度（10-25秒/操作）
- 第二次运行：**1-3秒/操作**（提升 5-10 倍）

**使用方法**：
```bash
# 第一次运行（生成缓存）
npx tsx demo-wind-robust.ts

# 第二次运行（使用缓存，速度极快）
npx tsx demo-wind-robust.ts
```

---

### 方案 2：使用更快的模型 ⭐⭐⭐⭐

**当前模型**：`huihui_ai/qwen3-vl-abliterated:4b-instruct`（4B 参数）

**推荐模型**：

| 模型 | 参数量 | 速度 | 准确率 | 推荐度 |
|------|--------|------|--------|--------|
| `qwen2-vl:2b` | 2B | ⚡⚡⚡⚡⚡ | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| `qwen2-vl:7b` | 7B | ⚡⚡⚡ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| `qwen2-vl:72b` | 72B | ⚡ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |

**切换模型**：
```bash
# 下载更快的 2B 模型
ollama pull qwen2-vl:2b

# 修改 .env
MIDSCENE_MODEL_NAME="qwen2-vl:2b"
```

**效果**：AI 分析时间从 5-15秒 降至 2-5秒

---

### 方案 3：减少等待时间 ⭐⭐⭐

**当前配置**：
```typescript
wait: {
  short: 1500,   // 1.5秒
  medium: 3000,  // 3秒
  long: 5000,    // 5秒
}
```

**优化配置**：
```typescript
wait: {
  short: 500,    // 0.5秒（减少 1秒）
  medium: 1500,  // 1.5秒（减少 1.5秒）
  long: 3000,    // 3秒（减少 2秒）
}
```

**效果**：总耗时减少 10-20%

**注意**：如果界面响应慢，可能导致操作失败

---

### 方案 4：减少重试次数 ⭐⭐

**当前配置**：
```typescript
retry: {
  maxAttempts: 3,
  delay: 3000,
}
```

**优化配置**：
```typescript
retry: {
  maxAttempts: 2,  // 减少到 2 次
  delay: 2000,     // 减少到 2 秒
}
```

**效果**：失败时节省 3 秒

**注意**：可能降低成功率

---

### 方案 5：使用录制回放模式（最快）⭐⭐⭐⭐⭐

**原理**：首次运行录制所有操作，后续运行直接回放，无需 AI。

**使用方法**：
```bash
# 第一次运行（录制模式）
npx tsx demo-wind-robust.ts

# 第二次运行（回放模式，无需 AI）
npx tsx demo-wind-robust.ts
```

**效果**：
- 首次运行：正常速度
- 第二次运行：**0.5-1秒/操作**（提升 10-20 倍）

## 💡 常见问题

### Q1: 为什么找输入框要几分钟？

**原因**：
- AI 模型分析图像慢（5-15秒）
- 重试多次（最多 3 次）
- 每次重试等待 3 秒

**解决**：
- 启用缓存（方案 1）
- 使用更快的模型（方案 2）

### Q2: 为什么输入几个字要等几分钟？

**原因**：
- Midscene 逐字输入，每个字都是一次 AI 操作
- 输入"股票数据浏览器" = 7 个字 = 7 次 AI 调用

**解决**：
- 启用缓存（方案 1）
- 使用录制回放模式（方案 5）

### Q3: 缓存为什么不生效？

**检查**：
```bash
# 确认 .env 配置
cat .env | grep CACHE

# 应该看到：
MIDSCENE_CACHE=true
MIDSCENE_CACHE_MODE=read-write
```

**可能原因**：
- 缓存文件被删除
- 界面发生变化，缓存失效
- 缓存模式设置为 `read` 而不是 `read-write`
